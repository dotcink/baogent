# AGENTS.md

## 项目目标

从 0 开始，逐步构建一个高度自主的 Agent：模型是内层智能，外围系统负责提供能力与约束，也就是 Harness。

## 分层

- `src/model`：只放模型与 provider 适配，不放 agent loop、工具编排或 CLI 逻辑。
- `src/agent`：负责 agent loop、工具调用与 bash 能力，不依赖 `src/entry`。
- `src/entry`：只做 CLI、配置加载与装配，保持薄。

## 规则

- 优先最简单、最直接的方案，避免无关重构。
- 优先复用现有抽象，不随意新增临时协议或重复封装。
- 保持单向依赖，不打破分层。
- 命名直接反映职责，不使用职责模糊的名称。
- 涉及 CLI、配置、默认值、文件名、环境变量的变更，必须同步更新代码、帮助文本、文档和示例。
- 新增或修改配置项时，必须同步更新 `config/example.toml`。

## 校验

- 修改后运行匹配范围的校验；如果没有更窄的检查，至少运行 `bun run typecheck`。
